# イシュー管理サマリー

このドキュメントは、voice-in（ローカル音声入力アプリ）のイシュー（課題・機能改修・バグ修正）を一元管理するためのインデックスです。

---

## カテゴリ定義

| カテゴリID | カテゴリ名 | 説明 | 例 |
|:---|:---|:---|:---|
| `APP` | App/Core | アプリの中核ロジック（録音、AI処理、貼り付け、設定ロード等） | 録音方式変更、例外制御 |
| `UI` | User Interface | 画面レイアウト、表示部品、見た目 | ダイアログ追加、メニュー追加 |
| `UX` | User Experience | 初心者向け導線、操作フロー、分かりやすさ | 初期設定ウィザード、エラー案内 |
| `PERF` | Performance | パフォーマンス改善、メモリ/CPU削減 | 録音の省メモリ化 |
| `HIST` | History | 文字起こし履歴の保存/参照 | 履歴50件保持、検索 |
| `SEC` | Security | APIキー管理、機密情報取り扱い | ログ/履歴にキーを残さない |
| `BUG` | Bug Fix | 不具合修正 | キー操作、貼り付け失敗 |
| `REFAC` | Refactoring | 機能変更なしの整理 | main.py分割 |
| `DOC` | Documentation | README/手順書の更新 | 初期セットアップ手順 |
| `QA` | Quality | テスト、回帰確認、ビルド手順 | pyinstaller確認 |
| `SYNC` | Integration | 複数機能を跨ぐ統合、干渉防止 | ウィザード×履歴×設定の統合 |

## ステータス定義

| ステータス | アイコン | 説明 |
|:---|:---|:---|
| 対応待ち | ⏳ | 調査・実装待ち |
| 対応中 | 🔄 | 現在作業中 |
| レビュー待ち | 👀 | 実装完了、レビュー待ち |
| テスト待ち | 🧪 | レビュー完了、テスト待ち |
| 完了 | ✅ | すべて完了 |
| 保留 | ⏸️ | 一時保留 |
| 却下 | ❌ | 対応しない判断 |

---

## 優先度定義

| 優先度 | ラベル | 説明 |
|:---|:---|:---|
| P0 | 🔴 Critical | 先に統合設計が必要（事故防止） |
| P1 | 🟠 High | 直近で体験に直結する改善 |
| P2 | 🟡 Medium | 早めに改善したい |
| P3 | 🟢 Low | 余裕があれば |
| P4 | ⚪ Backlog | 将来検討 |

## 新規イシュー作成手順

1. [ISSUE_TEMPLATE.md](./ISSUE_TEMPLATE.md) をコピー
2. 適切なカテゴリIDと連番でリネーム
3. テンプレートに従って内容を記載
4. 本サマリーの「オープン」セクションに追記
